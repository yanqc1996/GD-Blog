# vuepress+github搭建自己的博客

# 前言

本文基于_1小时搞定vuepress快速制作vue文档/博客+免费部署预览一文_结合自己的摸索_，_旨在解决项目结构及创建仓库等问题
vuepress官网：https://www.vuepress.cn/
代码仓库地址：https://github.com/aol121/wzp
博客预览地址：https://aol121.github.io/wzp/

# 前期准备

1.全局安装vuepress
   wind键+R键，输入cmd 回车 打开命令行工具，输入 npm install -g vuepress
2.github上创建仓库
    登录你的github账号，点击New repository创建新仓库，仓库名称随意，没有具体要求。
3.git clone ｛刚创建的仓库地址｝ ， 将刚创建的仓库克隆下来（空白文件夹）。
4.克隆或下载上文代码仓库的代码，将下载好的代码复制到步骤3克隆下来的空文件夹中。
5.参照下文做一些配置调整
6.提交至远程仓库。

dest设置为'./docs'，为后续github仓库设置GitHub Pages的Source做准备的
base设置根路径，此处的 wzp 替换成你自己创建的github仓库名称，否则会出现css等资源加载路基错误的问题

# docs文件夹

运行npm run build后打包出来的文件夹。

# github仓库设置

点击博客仓库的Settings，向下滚动至GitHub Pages，Source的下来列表选择第二个选项master branch/docs floder
Source上面一行浅蓝色底文字是你设置好以后的博客地址，类似：Your site is published at https://aol121.github.io/wzp/

# colorfully egg

为提升网站趣味性，添加了live2d元素，
主要思路是页面加载完成后创建canvas标签添加至body，然后调用loadlive2d（）
当前的引入方案仅提供参考，不一定是最佳方案，如果有更好的可以分享至评论区，大家共同学习
关于live2d的教程分享张鑫旭的文章二次元live2d看板娘效果中的web前端技术，他的其他文章也非常推荐大家阅读

文件路径为src/.vuepress/public/live2dModels/live2d
为保证在各自项目都能运行需要对init.js作一丢丢的调整---第八行 wzp 替换为你的仓库名
具体路径为src/.vuepress/public/live2dModels/live2d/js/init.js

window.addEventListener('load',function(){
  var canvas = document.createElement('canvas');
  canvas.id = 'live2d';
  canvas.width = '280';
  canvas.height = '250';
  canvas.style.cssText = 'position: fixed;right:-40px;bottom:0;';
  document.body.appendChild(canvas);
  loadlive2d("live2d", "/wzp/live2dModels/live2d/model/kesshouban/model.json");//wzp替换为仓库名
})

# 注意事项

本地启动开发不支持热更新
src中的代码调整完后先npm run build打包，再提交至远程仓库，博客内容才会更新
人生不如意事七八九，在实现自己的博客过程中难免会遇到些磕磕绊绊，耐心些仔细些终究是可以成功的。
